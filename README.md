# 🐾 Groomer Mini App — демо

Telegram Mini App для грумінг-салону. Без онлайн-бронювання — фокус на трьох речах:

1. **Самостійна анкета клієнта** — власник собаки сам заповнює інформацію через посилання.
2. **Автоматичні нагадування в Telegram** — за день до візиту, на день народження, нагадування про наступну стрижку.
3. **Картки клієнтів з історією** — для кожного: контакти, алергії, улюблений тип стрижки, журнал візитів.

## ▶ Швидкий старт (локально)

```bash
cd "Groomer demo"
python3 -m venv .venv
source .venv/bin/activate       # Windows: .venv\Scripts\activate
pip install -r requirements.txt
python demo.py
```

Відкрийте у браузері:

- **Грумер** — http://localhost:5050/app?role=groomer
- **Клієнт** — http://localhost:5050/app?role=owner
- **Анкета з нуля** — http://localhost:5050/app?role=owner&new=1

> Перший запуск автоматично створює `salon.db` і додає 4 фіктивні клієнти, щоб демо мало живий вигляд.

## 🤖 Підключити справжнього Telegram-бота

1. Створити бота через **@BotFather** → отримати токен.
2. Скопіювати `.env.example` → `.env` і вставити токен:
   ```
   TELEGRAM_BOT_TOKEN=123456:AA...
   SALON_NAME=Pawfect Studio
   SALON_ADDRESS=ul. Kwiatowa 12, Warszawa
   ```
3. Зробити сервер доступним з інтернету (для Mini App Telegram потребує HTTPS):
   ```bash
   # найшвидший варіант — ngrok
   ngrok http 5050
   # копіюємо https://xxx.ngrok-free.app
   ```
4. У `@BotFather` → `/mybots` → бот → `Bot Settings` → `Menu Button` → вставити URL `https://xxx.ngrok-free.app/app`.
5. Знову запустити `python demo.py` — і написати боту `/start`.

## 📨 Як працюють нагадування

Планувальник запускається щодня о **09:00** і перевіряє:

| Тип | Коли надсилається | Текст |
|---|---|---|
| 📅 Візит | За 1 день до запланованого візиту | "Завтра о 15:00 чекаємо {pet}…" |
| 🎂 День народження | У день народження собаки | "Сьогодні {pet} святкує…" |
| 🐶 Час нагадати | Через N тижнів (за замовч. 6) після останнього візиту | "Час на нову стрижку?" |

У дашборді грумера є кнопка **"📨 Нагадування"** — можна побачити попередній перегляд або надіслати одразу.

Якщо токен бота не заданий — нагадування показуються як попередній перегляд у UI (демо-режим).

### Як протестувати нагадування швидко

У `demo.py` знайдіть рядок:
```python
scheduler.add_job(dispatch_reminders, "cron", hour=9, minute=0, ...)
```
Тимчасово замініть на:
```python
scheduler.add_job(dispatch_reminders, "interval", seconds=30, ...)
```
Та запустіть знову — нагадування полетять кожні 30 секунд.

## 🚀 Деплой на сервер

### Які варіанти підходять / не підходять

| Платформа | Підходить? | Чому |
|---|---|---|
| 🟢 **Railway / Render / Fly.io** | ✅ Так | Простий деплой Python-додатка, постійний диск або Postgres, цілодобовий процес. **Рекомендую.** |
| 🟢 **VPS (Hetzner / DigitalOcean)** | ✅ Так | Повний контроль, найдешевше у довгостроковій. 4–5 €/міс. |
| 🟡 **Heroku** | ⚠ Умовно | Файлова система **ефемерна** — SQLite-база зникатиме при перезапусках dyno. Потрібно перейти на Postgres (треба переробити код). Безкоштовного тарифу більше немає. |
| 🔴 **Vercel** | ❌ Не підходить | Vercel — це **serverless**, він не вміє тримати фоновий процес (бот через polling + APScheduler не запрацюють). Підійшов би тільки якщо переробити бота на webhooks + винести нагадування в окремий cron-сервіс. |

### Деплой на Railway / Render (рекомендований шлях)

1. Створити репозиторій на GitHub з цим кодом.
2. Зайти на [railway.app](https://railway.app) (або [render.com](https://render.com)) → "New project" → "Deploy from GitHub".
3. Додати змінні середовища: `TELEGRAM_BOT_TOKEN`, `SALON_NAME`, `SALON_ADDRESS`, `GROOMER_TELEGRAM`.
4. Railway автоматично знаходить `Procfile` та запускає `python demo.py`. Render — у налаштуваннях Start Command: `python demo.py`.
5. Railway/Render видають HTTPS URL — вставляєте його в `@BotFather` як Menu Button.

> ⚠ Перед продакшеном переключитись з SQLite на Postgres: на Railway/Render файлова система переживає рестарти, але один-два розгортання можуть зкинути дані. Для серйозного використання — підключити Postgres (вони дають вбудовану БД).

### Деплой на Heroku (якщо вже звикли)

```bash
heroku create groomer-demo
heroku config:set TELEGRAM_BOT_TOKEN=... SALON_NAME=... SALON_ADDRESS=...
git push heroku main
```
`Procfile` і `runtime.txt` уже додані в проекті.

⚠ **Важливо для Heroku:** додайте addon Postgres і замініть SQLite на нього у `demo.py`, інакше всі дані зникнуть при будь-якому перезапуску. Це не маленька зміна — якщо плануєте Heroku, краще одразу проект перевести на Postgres + asyncpg (можу зробити в окремій ітерації).

### Чому Vercel взагалі не варто розглядати

Vercel виконує функції на запит (serverless). Telegram-бот через `long polling` потребує постійно запущеного процесу. Так само щоденний планувальник (APScheduler) не виживе між запитами. Альтернатива — переписати бота на webhooks + щоденний cron у GitHub Actions, але це додасть багато складності без вигоди порівняно з Railway за 5$/міс.

## 🧭 Шлях користувача (UX)

### Клієнт (власник собаки)
1. Перший раз: тисне кнопку в боті → Mini App → коротка анкета (8 кроків, по 1 питанню) → одразу бачить картку улюбленця.
2. Наступний раз: відкриває Mini App — бачить картку, банер з наступним візитом, історію.
3. Отримує нагадування в Telegram — без жодних дій з боку грумера.

### Грумер (Анна)
1. Відкриває Mini App → дашборд: список усіх клієнтів зі статус-чіпами (день народження, алергії, давно не був).
2. Фільтр по табах: Усі / Сьогодні / Дні народження / Час нагадати.
3. Тапає на клієнта → бачить повну картку: контакти, алергії, історія візитів.
4. Після стрижки: "+ Додати візит" → що робили, ціна, нотатки → автоматично планується наступне нагадування.
5. Якщо потрібно — "📅 Запланувати" наступний візит з конкретною датою/часом.
6. Або "📨 Нагадування" — надіслати одразу або переглянути текст.

- Експорт у CSV для бухгалтерії
